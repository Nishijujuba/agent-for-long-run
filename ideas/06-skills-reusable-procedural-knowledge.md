# 06 可复用资产：Skill 把一次成功变成可复用能力

## 状态

confirmed

## 一句话主张

Skill 是把团队经验产品化的容器：把提问方式、模板、脚本、检查命令和参考资料打包成按需加载能力，让长程任务在更少 token、更稳定格式和更清楚验收标准下复用。

## 适用角色

跨角色：数据报告模板、税务口径清单、代码 PRD/QA/Issue 生成、实验复盘模板、运维 runbook。

## 工作场景

团队第一次用 agent 完成一条 PRD 到 issue 的流程，期间沉淀了好问题、好模板和验证命令。下一次类似任务，团队直接调用 skill，减少反复粘贴长 prompt 的成本。

## Agent 协作方式

人类负责确认 SOP 与质量门槛；agent 负责调用 skill、按模板产出、执行脚本、补齐待确认项。Skill 像“团队方法包”，把一次高质量实践变成可维护资产。

## 长程任务设计要点

- `skill.md` 只放入口、适用场景、触发条件和关键步骤。
- 模板承载稳定结构：PRD、QA checklist、Issue、复盘报告、来源引用。
- 脚本承载确定性动作：提取、检查、验证、格式化。
- 采用渐进性披露：默认加载少量入口信息，需要时再展开细节。
- Skill 需要版本和维护责任，避免旧 SOP 混入当前任务。

## 风险与边界

Skill 复用会带来稳定性，也会带来过期风险。模板、脚本和 SOP 必须定期检查；涉及法规、生产环境、客户数据、权限操作的 skill 需要写清适用范围、禁用范围和人工审批点。

## PPT 使用建议

**页面标题**：Skill：把一次成功变成可复用能力

**页面主文案**：

- 入口：`skill.md`。
- 模板：PRD / QA / Issue。
- 脚本：检查 / 生成 / 提取。
- 产物：稳定可验收。
- 扩充未训练知识：团队口径、领域模板、工具命令。
- 渐进性披露：默认只放入口，触发后再展开细节。
- 推荐技能包：Grill、Write PRD、PRD-to-Issues、Architecture、QA Review、Source Citation。

**讲法**：Skill 像团队的“流程卡加工具箱”。一条流程跑通后，把问题清单、文档模板、验证命令沉淀下来，后续任务就能用更少上下文获得更稳定结果。
