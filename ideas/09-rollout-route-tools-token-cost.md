# 09 落地路线：工具、Token 与成本的生产节奏

## 状态

confirmed

## 一句话主张

长程 agent 落地应先用人类和旗舰模型跑通首个高价值任务，暴露未知环节；随后沉淀 skill、脚本、hook 和 harness，把低风险 AFK issue 交给成本更可控的执行链路，并把新错误写回检查系统。

## 适用角色

跨角色：希望把 agent 从个人试用推进到团队稳定流程的知识工作者和团队负责人。

## 工作场景

团队选择一个高频长程任务，例如“每周政策口径更新”“数据报告生成”“代码需求到 PR”“实验结果复盘”“运维告警归因”。第一次由人类深度参与，用最强模型把链路跑完整；第二次开始沉淀模板、脚本和检查点；第三次把边界清楚的部分交给更便宜的执行模型或自动循环。

## Agent 协作方式

人类负责选择任务、设定收益目标、审核风险和升级范围。Agent 负责执行已定义流程、产出中间结果、记录状态、触发验证。工具负责检索、测试、日志、成本统计和 trace。

## 长程任务设计要点

- 第一步：人 + 旗舰模型跑通首个任务，把未知环节一次性暴露出来。
- 第二步：沉淀 skill、脚本、hook、harness，把成功路径变成可重复资产。
- 第三步：让次旗舰模型或自动循环执行常规任务，优先选择低风险、边界清楚的 AFK issue。
- 第四步：把新犯错点补进测试、文档、脚本、规则和状态模板。
- 生产工具接入原则：只读优先、沙箱或 staging、最小操作范围、日志审计、敏感配置留在受控系统里。
- Token 效率原则：只把当前任务需要的信息带入上下文，其余信息留在文件、索引和工具里。

## 风险与边界

初始阶段就接入高影响工具会放大事故半径。初始阶段就追求低成本模型会掩盖流程缺口。更稳的路线是先用较高成本把流程跑通，再把稳定部分产品化，最后逐步降低执行成本和扩大自动化范围。

## PPT 使用建议

**页面标题**：落地路线：工具、Token 与成本

**页面主文案**：

1. 人 + 旗舰模型跑通首个任务：把未知环节一次性暴露出来。
2. 沉淀 skill / 脚本 / hook / harness：把成功路径变成可重复资产。
3. 次旗舰模型执行常规任务：先放低风险、边界清楚的 AFK issue。
4. 新犯错点补进检查系统：错误进入测试、文档、脚本、规则。

**生产工具接入原则**：只读优先、沙箱或 staging、最小操作范围、日志审计、敏感配置留在受控系统里。

**下周行动**：选一个高频长程任务，切成 1 个对齐段 + 3 个 AFK issues。

**讲法**：落地路线像修一条生产线。第一件产品可以贵一点，因为目标是暴露流程问题；后面靠 skill、hook、测试和状态文件把成本降下来。
