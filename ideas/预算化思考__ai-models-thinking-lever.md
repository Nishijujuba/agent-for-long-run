# 把 Agent 的思考变成预算化资源

## 状态

candidate

## 一句话主张

长程任务里，Agent 的“思考”应被当成可分配的工作预算：先设质量线、时间、成本、工具和复核边界，再让 Agent 在这些边界内决定何时思考、何时调用工具、何时交付结果。

## 适用角色

跨角色：税务管理员、程序员、数据分析师、算法工程师、平台运营工程师，以及需要让 agent 连续处理复杂材料的一线知识工作者。

## 工作场景

一个税务管理员让 agent 完成“从多份政策资料中形成风险材料清单”的任务：检索政策、整理口径、标注冲突条款、生成材料清单、列出待人工确认的高风险判断。类似方法也可迁移到代码修复与测试、数据分析报告、论文调研、线上工单归因。

## Agent 协作方式

人负责设定目标、质量线和边界：什么算完成，最长等多久，允许花多少 token 或费用，能调用哪些工具，哪些结论必须人工复核。

Agent 负责在任务过程中分配资源：简单步骤快速处理；证据冲突时增加推理；需要外部信息时调用工具；风险较高的节点保留检查点并向人追问。

直观类比：思考空间像草稿纸，努力等级像油门，任务预算像油箱、限速牌和项目工时表。草稿纸太少会让复杂题仓促交卷，油门踩到底会消耗过大，预算的作用是把资源送到最该花的位置。

## 长程任务设计要点

- 目标：写清任务产物，例如“形成可审计的政策口径表”或“提交通过测试的代码修复”。
- 资料：列出允许使用的文件、网页、数据库、代码库或业务系统。
- 计划：把任务拆成检索、分析、生成、验证、交付几个阶段。
- 检查点：在资料读完、初步结论形成、工具执行后、最终交付前设置停顿。
- 记忆：把关键结论、已排除路径、风险点写入文件或中间摘要，防止长任务丢上下文。
- 工具：为搜索、读写文件、运行测试、查询系统设置权限边界。
- 验收标准：用质量线表达最低要求，例如 \(Q(E) \ge Q_{min}\)。质量 \(Q\) 可近似理解为：

\[
Q \approx f(M, E, B_{tok}, task, context)
\]

其中 \(M\) 是模型，\(E\) 是努力等级，\(B_{tok}\) 是 token 预算，`task` 是任务难度，`context` 是上下文。

- 任务预算：把预算扩展为：

\[
B_{task}=B_{tok}+B_{time}+B_{cost}+B_{tool}+B_{review}
\]

它覆盖 token、时间、费用、工具调用和复核资源。

## 风险与边界

预算化思考不能替代业务责任。税务、合规、隐私、生产系统操作和客户影响较大的结论，仍需要人类最终判断。

低努力适合分类、抽取、格式化等结构清楚的任务；复杂推理、代码修改、政策解释和根因分析需要更强模型、更高努力等级或更多复核。

最高努力不应成为日常默认。PDF 指出高档位附近可能出现边际收益递减，继续加 token 可能只带来很小质量提升，却显著增加成本和延迟。

如果上下文本身错误或资料缺失，更多思考会围绕错误信息打转。预算化任务必须同时管理资料来源、工具权限和检查点。

## PPT 使用建议

适合放在“长程任务方法”页，作为一个跨角色通用框架。

页面主图建议做成一个“任务控制面板”：左侧是人设定的质量线、时间、成本、工具、复核；右侧是 agent 在任务中选择思考、调用工具、写输出；底部用一句话收束：人设标准和预算，agent 分配计算与行动。

可配例子：让 agent 做一份政策风险材料清单。低风险格式整理用低努力；法规冲突判断用高努力；引用来源必须可追溯；最终判断由税务管理员确认。
